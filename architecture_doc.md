# LILA APM v2 Architecture

## 1. System Overview
LILA APM is a web-based visualization tool that reads telemetry payload files (Parquet/Nakama), processes them into static JSON state files, and serves them via a Flask backend to a vanilla Javascript frontend canvas engine.

## 2. Component Design

### 2.1 Backend / Ingestion (ETL)
- **`server.py` (Flask Server)**
  - Exposes port `8000`.
  - Serves static files from `/public`.
  - Endpoint `/upload` accepts multi-part form data containing `.nakama-0` (Parquet) files, saving them to `player_data/Uploaded/`.
  - Triggers the parsing script.
- **`parser.py` (Data Parser)**
  - Reads Parquet file blocks using `pyarrow` and `pandas`.
  - Parses the `ts` schema column purely as **Elapsed Match Time (Milliseconds)**.
  - Generates lightweight JSON files in `/public/data/` organized by Map name.

### 2.2 Frontend Engine (`public/app.js`)
- **State Store (`appState`)**
  - Holds the raw `data` blob generated by the backend.
  - Maintains `FilteredTimeline`: an array of events strictly complying with the current UI constraints (Date, Map, User, Time boundaries).
- **Time Filter Logic**
  - **Relative Constraints**: Slices the overall timeline to the specific window (e.g. 0 elapsed ms to 300,000 elapsed ms).
  - **Absolute Constraints**: Compares literal Wall-Clock timestamp thresholds against the match events. 
  - Events occurring strictly outside these slices are destroyed from the viewport context, ensuring the canvas never renders impossible future movements or unrequested history.
- **Render Engine**
  - Synchronizes physical render loop with the user scrubber.
  - Loops sequentially from Event 0 up to `Scrubber.CurrentEvent`.
  - Uses native CSS Canvas APIs; Heatmaps utilize `lighter` compositing with custom radial gradients for density accuracy.
